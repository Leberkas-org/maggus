# Bug: Repos/config/prompt views not full size; update view flickers on menu navigation

## Summary

Three TUI screens (repos, config, prompt) render a box that is centered instead of filling the terminal, leaving blank margins. The update view flickers when navigating to/from the main menu because it runs as a subprocess via `tea.ExecProcess` rather than as an integrated app-router screen.

## Steps to Reproduce

**Bug A – Views not full size:**
1. Run `maggus` to open the main menu
2. Navigate to Repos, Config, or Prompt
3. Observe: the bordered box does not fill the terminal; margins appear on all sides

**Bug B – Update view flickering:**
1. Run `maggus` to open the main menu
2. Navigate to Update
3. After the update finishes, navigate back to the main menu
4. Observe: the terminal flickers / blanks briefly when transitioning

## Expected Behavior

- Repos, config, and prompt views should fill the entire terminal (no visible margins)
- Navigating to/from update should be seamless with no flicker, consistent with other sub-screens

## Root Cause

### Bug A – Centering instead of filling

`FullScreen`, `FullScreenLeft`, and `FullScreenLeftColor` in `src/internal/tui/styles/styles.go` (lines 139, 175) all call:

```go
return lipgloss.Place(width, height, lipgloss.Center, lipgloss.Center, box)
```

The `box` itself is sized to `innerW + 2` / `innerH`, which correctly fills the available area. But `lipgloss.Place` with `Center/Center` **pads the box to be centered** within the terminal rather than placing it flush at top-left. This creates visible margins.

Affected call sites:
- `src/cmd/repos.go:543,566` — calls `FullScreenLeftColor`
- `src/cmd/config_view.go:159` — calls `FullScreenColor`
- `src/cmd/prompt_picker.go:297` — calls `FullScreenColor`

### Bug B – Update runs as subprocess, not integrated screen

`src/cmd/menu_update.go:284–292` dispatches update as an `execProcessMsg`:

```go
return m, func() tea.Msg {
    execPath, _ := os.Executable()
    return execProcessMsg{
        cmd:    exec.Command(execPath, "update"),
        onDone: func(error) tea.Msg { return navigateBackMsg{} },
    }
}
```

All other major screens (status, config, repos, prompt) use `navigateToMsg` and are integrated into the app router (`src/cmd/app_model.go:19–25`). The update command still spawns a separate `tea.NewProgram` subprocess, which causes an alt-screen tear-down and reinitialisation on return, producing the flicker.
