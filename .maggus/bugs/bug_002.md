# Bug: TUI status view shows all plans as unapproved on initial render

## Summary

When opening the TUI status view (`maggus status`), all features and bugs display the `○` unapproved badge regardless of their actual approval state. The approval state is correct on disk and plain mode (`--plain`) renders it correctly — only the TUI initial render is affected.

## Steps to Reproduce

1. Approve a feature or bug: `maggus approve <id>`
2. Verify `feature_approvals.yml` shows `true` for that plan's UUID
3. Run `maggus status` (TUI mode, no flags)
4. Observe: all plans show `○` (orange unapproved badge) in the left pane

## Expected Behavior

Approved plans should show `✓` (green) on initial render without requiring a file change to trigger the watcher.

## Root Cause

In `src/cmd/status_cmd.go:185`, both `features` and `approvals` are loaded:

```go
features, approvals, err := loadPlansWithApprovals(dir, true)
```

But at line 222, only `features` is passed to `newStatusModel` — `approvals` is dropped:

```go
m := newStatusModel(features, all, nextTaskID, nextTaskFile, agentName, dir, showLog, approvalRequired)
```

`newStatusModel` (at `src/cmd/status_model.go:90`) never accepts or sets the approvals map, so `m.approvals` starts as `nil`.

When `renderLeftPane` calls `isPlanApproved(plan, m.approvals, m.approvalRequired)` and approval is required (opt-in mode), `approval.IsApproved(nil, featureID, true)` evaluates `nil[featureID]` which returns the zero value `false` — making every plan appear unapproved.

`reloadPlans()` does correctly load approvals (`m.approvals = a` at `status_model.go:159`), but it is only triggered by the file watcher, not during initialization. So approvals never load unless a file change occurs while the TUI is open.

## User Stories

### BUG-002-001: Pass initial approvals into newStatusModel

**Description:** As a user, I want the TUI status view to show correct approval badges from the moment it opens so I can see which plans are approved without having to trigger a file reload.

- [ ] On initial render, approved plans display `✓` (green) and unapproved plans display `○` (orange)
- [ ] Subsequent file watcher reloads via `reloadPlans()` continue to work correctly
- [ ] No regression in plain mode (`--plain`) approval rendering
