# Bug: Stop-after-task indicator not shown when navigating to status view

## Summary

When the stop-after-task sentinel is active and the user navigates to the status view, neither the footer ("⏳ Stopping after task…") nor the left-pane indicator is displayed. The state only appears after the next daemon cache update event fires.

## Steps to Reproduce

1. Start `maggus daemon` with a running task
2. Activate stop-after-task (e.g. via `maggus stop --after-task` or the CLI sentinel file)
3. Open `maggus status` (navigate to it from the menu)
4. Observe: the status footer shows the normal hint bar, not "⏳ Stopping after task…"
5. Observe: the left pane does not show the stopping indicator

## Expected Behavior

The "⏳ Stopping after task…" footer and left-pane indicator should be visible immediately upon entering the status view when the sentinel file is present.

## Root Cause

`buildStatusModel()` at `src/cmd/app_model.go:391–395` seeds the initial daemon state from the cache, but omits `StoppingAfterTask`:

```go
if daemonCache != nil {
    cached := daemonCache.Get()
    sm.daemon.PID = cached.PID
    sm.daemon.Running = cached.Running
    // sm.daemonStoppingAfterTask is NOT seeded here
}
```

`daemonStoppingAfterTask` therefore starts as `false` regardless of what the sentinel file says. It only becomes `true` when the next `daemonCacheUpdateMsg` arrives — which only fires when the daemon state files change (i.e. when the task finishes and the daemon exits). Until that event, the status view renders as if stop-after-task were not set.

The analogous seed for `Running` and `PID` is already present; `StoppingAfterTask` simply needs to be added alongside it.
