# Bug: Remove auto_work config — menu should always auto-start when approved work exists

## Summary

The `auto_work` config setting (`disabled`/`enabled`/`delayed`) is unnecessary now that daemon mode is the primary workflow. When approved workable tasks exist, maggus should always start working immediately — both on menu launch and when new files appear. The setting, its config UI row, and the delayed countdown logic should be removed.

## Root Cause

The `auto_work` setting was introduced before daemon mode made the menu the primary interface. Now it adds a configuration step that has no reason to exist: if a user has approved tasks, the intent is clearly to work on them.
