# Bug: Discord presence setting is per-repo but should be global

## Summary

The `discord_presence` setting lives in `.maggus/config.yml` (per-repository) but Discord Rich Presence is a user/machine preference that has nothing to do with individual repositories. Users must configure it separately in every repo, and the config TUI shows it under the Project tab instead of Global.

## Steps to Reproduce

1. Run `maggus config` in repository A
2. Enable Discord presence under the Project tab
3. Open repository B
4. Run `maggus config` — Discord presence is off
5. User must re-enable it for every repository

## Expected Behavior

Discord presence should be a global setting in `~/.maggus/config.yml`, configured once and applied everywhere. The config TUI should show it under the Global tab.

## Root Cause

The `DiscordPresence` field is defined on the per-repo `Config` struct at `src/internal/config/config.go:147`:

```go
DiscordPresence bool `yaml:"discord_presence"`
```

The global `Settings` struct at `src/internal/globalconfig/globalconfig.go:75-77` only contains `AutoUpdate` — there is no `DiscordPresence` field.

All consumers read from the per-repo config:
- `src/cmd/root.go:69` — `cfg.DiscordPresence` to init presence in menu
- `src/cmd/work.go:232` — `wc.cfg.DiscordPresence` to init presence in work loop
- `src/cmd/prompt.go:96` — `cfg.DiscordPresence` to init presence in prompt command

The config TUI at `src/cmd/config.go:192` places "Discord presence" in `projectRows` instead of `globalRows`, and `buildConfig()` at line 257 writes it back to the per-repo config.
