# Bug: Daemon does not wake up after approving a feature or bug

## Summary

When the daemon is idle (no workable tasks) and the user approves a feature or bug via `maggus approve`, the daemon stays idle instead of waking up to process the newly approved work. The file watcher only monitors `feature_*.md` and `bug_*.md` files but ignores changes to `feature_approvals.yml`.

## Related

- **Commit:** dbe5289 (fix(daemon): remove poll timer from wait loop, use pure fsnotify)
- **Commit:** 7606a24 (fix(daemon): persist filewatcher across wait cycles)

## Steps to Reproduce

1. Start `maggus work` (daemon mode) with approval required enabled in config
2. Add a feature file to `.maggus/features/` (daemon detects it but skips because unapproved)
3. Wait for daemon to enter idle/watching state ("no work found, watching for changes")
4. Run `maggus approve feature_001`
5. Observe: daemon remains idle and does not pick up the approved feature

## Expected Behavior

After approving a feature or bug, the daemon should immediately wake up, re-evaluate workable tasks, and start processing the newly approved work.

## Root Cause

The file watcher has two gaps that prevent it from detecting approval changes:

**1. Watched directories do not include `.maggus/` root**

`src/internal/filewatcher/filewatcher.go:44-47` — `New()` only watches two directories:

```go
dirs := []string{
    filepath.Join(baseDir, ".maggus", "features"),
    filepath.Join(baseDir, ".maggus", "bugs"),
}
```

The approval file lives at `.maggus/feature_approvals.yml`, which is in the `.maggus/` root directory — not watched at all by fsnotify.

**2. `isRelevantEvent()` rejects non-markdown files**

`src/internal/filewatcher/filewatcher.go:202-216` — Even if the `.maggus/` directory were watched, the event filter would reject it:

```go
func isRelevantEvent(event fsnotify.Event) bool {
    // ...
    name := filepath.Base(event.Name)
    if strings.HasPrefix(name, "feature_") && strings.HasSuffix(name, ".md") {
        return true
    }
    if strings.HasPrefix(name, "bug_") && strings.HasSuffix(name, ".md") {
        return true
    }
    return false
}
```

`feature_approvals.yml` fails both checks because it ends with `.yml`, not `.md`.

**Impact amplified by BUG-004-001:** Commit dbe5289 removed the polling fallback timer that previously served as a safety net. The daemon now relies entirely on fsnotify, making this gap a hard block rather than a delayed detection.
