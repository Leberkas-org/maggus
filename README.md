# Maggus

<img src="docs/avatar.png" alt="Maggus" width="300">

This will be your best and worst co-worker at the same time. A stupid junior developer that will just work and only ask questions when the work itself is unclear.

Provide him a `IMPLEMENTATION_PLAN.md` and he will work. To create this plan you can use the **iplan** skill.

### Planned Features

It should be a single binary that is available for Windows/Mac/Linux. Written in Go with the Cobra lib to get a Terraform like CLI. It will help you read the implementation plans and work on the listed user-stories/tasks and prompt then to the KI-Agent of your choice. in the MVP this will be Claude Code.

In later features a service should host the user stories so that the file is not needed anymore. It should be like a Jira board optimized for maggus to read, and for humans to edit, plan and supervise.
